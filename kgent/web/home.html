How does dropping an edge on empty canvas create a new node?
kgent · grounded in 4 passages

In Home.tsx, the onConnectEnd handler drives this. When a connection is released on the pane rather than a handle, it mints a fresh node:

  1. getId() returns the next sequential node id.
  2. screenToFlowPosition converts the cursor point into graph coordinates so the node lands under the pointer.
  3. setNodes appends the node and setEdges wires it back to the source.

The whole handler is wrapped in useCallback so React Flow keeps a stable reference across renders.

Why it isn't just a chatbot

Three ideas that make kgent an instrument.

01

Citations are coordinates

Each source in an answer is a node on the map. Hover a citation and its neighbourhood lights up, so you always see where a claim came from, not just that a claim was made.

02

Provenance you can open

Expand any answer to read the exact chunks, with document path and position. No hidden context window, no guesswork; the evidence is one click away.

03

Runs on your machine

Ollama and Chroma by default, and API keys stay in the browser. Point kgent at private code or a mailbox without a single byte leaving your laptop.
